Extraction Panel — Revised Design

Frame + Relation structure instead of sentence cards

Two display modes for the extraction panel

A

Graph Primary, YAML Secondary

Default view is the visual graph (clusters, relations, frame nodes). Toggle to Lite YAML for text editing. Best for visual thinkers who want to see relationships at a glance.

B

YAML Primary, Graph Secondary

Default view is the Lite YAML (readable, editable). Toggle to graph for spatial view. Best for developers who think in structured data and want direct editing.

C

Adaptive — Small Panel = YAML, Expanded = Graph

When panel is narrow (sidebar mode), show Lite YAML since it fits better. When panel is expanded wide (or full-screen), show the graph. User can always toggle. Best of both worlds.

Key Interactions

  • Live extraction — as conversation progresses, new frames appear with purple "NEW" badges. Existing frames update in-place.
  • Click frame (YAML or Graph) → highlights source text in chat with purple underline. Shows exact turn + char range.
  • Edit YAML directly → changes generate a user_yaml_edit delta, same pipeline as LLM extraction.
  • Edit Graph directly → drag to create relations, double-click to add frames, right-click to delete. All generate deltas.
  • Lineage on hover → hover a frame to see its history: which delta created it, which conversation turn, how many times updated.
  • Previous commits dimmed → frames from earlier commits shown with lower opacity. New/changed frames highlighted.
