Complete Design — Chat-First Onboarding

Full summary for approval before writing spec

1. First-Time User Experience

BEFORE (Current)
Login → Welcome Modal → Experience Dialog → Projects Page (empty) → Create Project → Empty Canvas → Add Conversation → Chat → Extract → Commit
6 steps before value
AFTER (New)
Login → Chat box → Type anything → See extraction → Commit
1 step before value

2. Landing Page (New User)

  • Clean chat interface — no modals, no tour, no experience dialog
  • Minimal sidebar (collapsed or slim) with just "New Chat" and settings
  • Chat box with subtle ghost suggestions (optional, non-intrusive)
  • Upload (📎) and paste (📋) buttons next to input
  • Project auto-created from first message topic
Removed: WelcomeModal, OnboardingDialog, GuidedTour, QuickStartChecklist — all replaced by the product teaching itself through use.

3. Chat Workspace Layout

Sidebar

conversations

ChatGPT-style conversation

Commit action at bottom
Context awareness: Chat header always shows which project + branch the user is chatting toward. Extraction lands on that branch.

4. Extraction Panel

Graph View (Default)
  • Frame nodes with relation arrows
  • Auto-clustering by connected components
  • New frames highlighted in purple with "NEW" badge
  • Interactive: drag relations, double-click to add, right-click to delete
  • Click frame → highlights source in chat
  • Semantic-aware ELK layout (per scalability spec)
YAML View (Toggle)
  • Lite YAML — no IDs, no confidence, no source refs
  • Frame types as top-level keys, slots as properties
  • Relations section at bottom
  • Directly editable — saves generate user_yaml_edit deltas
  • New frames highlighted with purple left border
  • Syntax-highlighted for readability

5. Returning User Experience

  • Session restore — app remembers last active project + conversation via cookie/localStorage. User lands directly in their working context.
  • Sidebar populated — shows their projects with conversations grouped inside. Recent conversations at top.
  • "New Chat" always accessible — top of sidebar, creates new conversation in current project or new project.
  • Canvas accessible per project — each project in sidebar has a "Canvas" entry to switch to the knowledge graph view.

6. Canvas = Control Room

Canvas is no longer the entry point — it's the management view you switch to when you need to:
See full knowledge lineage Branch & merge Run leaf assertions Review diffs Monitor runner status Compare commits
Accessed via: sidebar "Canvas" link per project, or "📊 Canvas →" button in chat header.
