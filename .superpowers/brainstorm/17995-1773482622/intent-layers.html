The Problem: Capture Everything vs. Show What Matters

T3X captures all the semantic points — but humans want to see "what do I want right now?", not a giant dump of every detail discussed.

Three approaches to solve this

How do we show "current intent" without losing detail?

A

UI Layering (no algorithm change)

Keep all frames, but use collapsible tiers in the UI:

  • Tier 1 — Key Decisions: Frames updated or confirmed in recent turns (high confidence, recently touched)
  • Tier 2 — Supporting Details: Older frames, lower confidence, logistics
  • Tier 3 — History: Removed/superseded frames (collapsed by default)

Sorting heuristic: recency × confidence × change_count. No LLM needed.

Pros

  • Pure UI — no core changes
  • Deterministic, fast
  • User controls what to expand

Cons

  • Heuristic may not match what "matters"
  • Can't truly summarize intent
B

LLM Summary Layer (optional plugin)

After each extraction, optionally ask the LLM: "Given all these frames, what are the user's top 3-5 current intentions?" Display this as a summary card above the detailed frames.

  • Summary Card: LLM-generated "Current Intent" (3-5 bullet points)
  • Below: Full frame list (same as today, scrollable)
  • Summary regenerates after each turn or on demand

Pros

  • Truly understands what matters (semantic, not heuristic)
  • Natural language — easy for humans
  • Aligns with T3X's "LLM as optional plugin" philosophy

Cons

  • Extra LLM call per turn (cost, latency)
  • Non-deterministic — summary may vary
  • Core must stay deterministic (summary is display-only)
C

Both: UI Tiers + Optional LLM Summary (Recommended)

Combine A and B. The deterministic tier system always works (no LLM needed). When an LLM is available, it generates a summary card that replaces the heuristic Tier 1.

  • Without LLM: Tier 1 = recently changed frames (sorted by recency × confidence)
  • With LLM: Tier 1 = "Current Intent" summary card (3-5 natural language bullets)
  • Always: Tier 2 = all frames (expandable), Tier 3 = history (collapsed)

Pros

  • Works without LLM (deterministic fallback)
  • Better with LLM (semantic understanding)
  • Matches T3X philosophy: core deterministic, LLM optional
  • Layered UI keeps it clean

Cons

  • More implementation work
  • Need to design summary prompt carefully
