Extraction Strategy: How should T3X treat conversation progression?

As a conversation progresses, the user's intent sharpens. How should the semantic model reflect this?

A

Capture All, Highlight Recent

Extract everything from every turn. The latest turn's frames get visual emphasis (bold border, "fresh" badge). Older frames fade slightly but remain visible. Nothing is lost.

Pros

  • Complete semantic history
  • User can scroll back to see evolution
  • No information loss

Cons

  • Panel gets cluttered over long conversations
  • Hard to see "what matters now"
B

Capture All, Show Confirmed Only

Extract everything into the delta log (full history preserved). But the graph/YAML only shows confirmed or latest-turn frames. Toggle to reveal "tentative" frames from earlier turns.

Pros

  • Clean default view — shows user's current direction
  • Full history still accessible via toggle/filter
  • Maps to your "confirmed vs uncertain" idea

Cons

  • Requires confidence/recency scoring logic
  • Toggle adds UI complexity
C

Progressive Snapshot (Recommended)

Capture all, but the extraction model is a living snapshot — each turn's delta updates the single current state. Added frames appear, updated frames change, removed frames disappear. The delta log preserves history. The graph always shows "where we are now."

This is actually what T3X already does — the delta system adds/updates/removes frames. The missing piece is visual indicators showing what just changed (green=new, yellow=updated, strikethrough=removed) and confidence styling for uncertain frames.

Pros

  • Already how the system works — minimal new logic
  • Graph always reflects current understanding
  • Delta log is the history layer
  • Confidence field already exists on Frame

Cons

  • Need to decide what "just changed" means visually

Your intuition mapped to these options:

"Can the latest answer be the direction people want?" → This is Option C — the snapshot always reflects the latest state.

"We still need to capture all" → Yes — the delta log preserves every change. Nothing is lost.

"Just address confirmed or most recent" → The graph shows current state. Confidence styling distinguishes confirmed (solid) from tentative (faded).
