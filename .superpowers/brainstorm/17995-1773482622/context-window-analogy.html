The "Context Window" Model for Semantic Extraction

Like how an LLM uses full context but weights recent turns — T3X should show the current understanding while preserving the full history

How it works: The Living Snapshot + History
Conversation
Turn 1: "I want to plan a trip to Japan in April, budget $5000"
Turn 2: "We're vegetarian, prefer ryokan in Kyoto"
Turn 3: "Actually, change budget to $7000 and add Osaka"
Turn 4 (latest): "Skip Kyoto, focus Tokyo + Osaka only"
Current Snapshot (Graph/YAML)
# What human wants RIGHT NOW
destination: [Tokyo, Osaka]      # updated turn 4
budget: $7000                    # updated turn 3
diet: vegetarian                 # from turn 2
~~kyoto_ryokan: removed~~         # removed turn 4
osaka_street_food: new           # added turn 4
Delta History (expandable)
Turn 1: +destination:Japan, +budget:$5000, +month:April
Turn 2: +diet:vegetarian, +accommodation:ryokan
Turn 3: ~budget:$5000→$7000, +destination:Osaka
Turn 4: -kyoto_ryokan, ~destination:[Tokyo,Osaka], +osaka_food

The analogy:

LLM Context Window T3X Extraction
Full conversation in context Delta log preserves every change
Answer reflects latest intent Snapshot shows current understanding
Earlier context still influences Old frames retained unless explicitly removed
User can refer back to earlier turns Delta history lets user see/restore old decisions
