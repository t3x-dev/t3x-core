Complete Design: Extraction Panel UX Enhancements

Everything we agreed on, in one view. Review and confirm before I write the spec.

1. Relevance Model — "What does the user want now?"

Frames are ranked by a pluggable relevance score. MVP ships with two deterministic signals:

  • Recency — when was this frame last added/updated? Recent = higher score
  • Human confirm — user clicked the checkbox = locked at top relevance

Architecture supports future signals (reference frequency, centrality, user interaction) without rearchitecting.

LLM re-rank toggle: A checkbox in the panel header ("Focus intent"). When on, asks the LLM to judge which frames matter to the user's current goal. Old-but-important frames (e.g., allergy from turn 2) get re-highlighted with a purple "LLM: core intent" badge. When off, pure deterministic ranking.

Display tiers:

  • Highlighted — high relevance (recent, confirmed, or LLM-boosted)
  • Faded — low relevance (old, unconfirmed, not LLM-selected)
  • History — removed frames (collapsed section, expandable)

2. Delta Visualization — "What just changed?"

After each turn's extraction, changes are visually indicated in both graph and YAML views:

Green = added this turn
Yellow = updated this turn
Red = removed this turn

Persistence: Indicators stay until the next extraction. A "Changes" badge in panel header (e.g., "+2 ~1 -1") allows on-demand review of last turn's changes.

YAML view: Left border color-coded per line. Removed lines shown with strikethrough.

Graph view: Node border color matches change type. Removed nodes shown dashed + faded.

3. Confidence Styling — "Is this confirmed or tentative?"

Each frame shows a confirm checkbox:

  • Confirmed (✓) — solid border, full opacity, checkbox checked. Confidence → 1.0. Feeds "human confirm" signal to relevance model.
  • Uncertain (☐) — dashed border, 70% opacity, checkbox unchecked. Frames from LLM questions start here (low confidence).

Clicking the checkbox is a single action that: sets confidence to 1.0, boosts relevance score, and changes visual style from dashed→solid.

The extraction LLM sets initial confidence based on the source turn — user statements get high confidence, LLM questions get low confidence.

4. Model Selector — "Which LLM is responding?"

Location: Header bar chip, between conversation title and branch badge.

Display: Purple chip showing current model (e.g., "⚡ Claude Sonnet 4.5 ▾").

Interaction: Click opens popover grouped by provider (Anthropic, OpenAI). Selecting a model updates the conversation and all subsequent chat/extraction calls use it.

Reuses: ConversationModelChip from PR #554, model catalog from core, /v1/chat/providers API.

5. Relation Edges — already working

FrameGraphMini already renders relation edges with labels. If the LLM extraction produces relations (causes, conditions, elaborates, etc.), they appear as edges. No UI changes needed — just ensure the extraction prompt requests relations.
