1. Architecture Database + DML

The YAML tree is a database. Each extraction is a DML transaction. The panel shows the transaction, not the database.

// The model
Full YAML Tree = database (background, LLM reads this)
Extraction = DML transaction (INSERT / UPDATE / DELETE)
Panel = shows the transaction, not the full database
Commit = persists transaction to hash chain
// Per extraction cycle
input: full_tree + conversation_since_last_commit
llm_output: YOps[] (add / set / drop / unset)
panel_shows: only the YOps, grouped as triage rows
on_commit: YOps applied to tree → new commit hash
// Panel never grows. Always shows this extraction's delta.
