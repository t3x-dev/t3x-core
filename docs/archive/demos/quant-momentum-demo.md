# Demo 演示手册：量化动量策略 — 从研究到可版本化语义资产

> **目标受众**：量化交易团队负责人
> **核心卖点**：t3x 不是帮你写策略，而是把策略的语义版本变成资产 — 可追溯、可复盘、可交接、可自动喂给回测/执行。
> **所需时间**：15-20 分钟完整演示

---

## 环境准备

### 1. 启动服务

```bash
# 终端 1：启动 API 服务

```

确保 `.env` 中已配置：

访问 `http://localhost:3000`，确认项目列表页面正常加载。

策略：美股大盘股动量策略，每周调仓一次。
标的池：S&P500 成分股；剔除最近 20 日平均成交额低于 2000 万美元的股票。

### 你展示什么

## Step 2：Commit V1（策略语义版本化）

### 你做什么

1. 在对话界面中，找到消息上方/旁边的 **Commit 入口**（对话节点上的操作按钮）
2. 系统会显示 **SelectableTextBlock**，自动从文本中提取语义条目（Sentences）
3. 检查提取的条目，确认关键语义被正确拆分
4. 点击 **Commit** 确认提交

### 你展示什么（最关键）

Commit 结果页 / 节点中能看到：

- **Commit 节点** 出现在 Canvas 上，连接到 Conversation 节点
- **Sentences（策略语义条目）** — 类似以下效果：
  - S1：策略频率 = 每周调仓
  - S2：标的池 = S&P500 成分股
  - S3：流动性过滤 = 20日均成交额 >= 2000万美金
  - S4：信号 = 60日收益率排序
  - S5：建仓 = Top10% 多 / Bottom10% 空
  - S6：权重 = 等权，但单票上限 2%
  - S7：目标波动率 = 12% 年化
  - S8：回撤触发 = 回撤>8% 降杠杆至0.5
  - S9：成本假设 = 10bp + 5bp
  - S10：风险提示 = 震荡市反转，需要市场状态过滤
- **SourceRef** — 每条 Sentence 可追溯到原文

### 你说什么

> "现在 V1 策略的语义已经被拆成可引用的条目，每条都有来源。"
> "这一步是量化团队最缺的：策略定义从 Word 变成可版本化资产。"

## Step 4：Commit V2 + 查看 Diff

### 你做什么

1. 再次在对话节点上点击 **Commit** 入口
2. 系统提取新一轮的语义条目，生成 **Commit V2**
3. Canvas 上出现第二个 Commit 节点，连接关系清晰
4. 点击两个 Commit 之间的 **Diff 功能**（在 Commit 侧边栏或节点操作菜单中）

### 你展示什么（超级关键）

**Semantic Diff 结果**：

| 分类 | 示例 |
|------|------|
| **MODIFIED** | 动量窗口：60日 → 120日 |
| **MODIFIED** | 空头腿：Bottom10% → Bottom5% |
| **ADDED** | 市场状态过滤：SPY 20日收益率 < 0 且 VIX > 25 时降仓 30% |
| **MODIFIED** | 回测结果：年化 12-18% → 11-16%，回撤 <15% → 13% |
| **SAME** | 标的池、信号排序方法、交易成本等未变项 |

- V2 是在 V1 基础上 **演进** 出来的（DAG 关系清晰）
- 变化点一目了然：哪些 Sentence 新增/修改/不变

### 你说什么

> "你看，策略迭代不是'改文档'，而是产生一个新语义版本。"
> "V1/V2 都能被复现，争论时不会各说各话。任何人打开这个项目，都能看到策略是怎么一步步演进的。"

## Step 5：创建 Leaf（策略研报 / Article）

### 你做什么

1. 在 Canvas 上，从 Commit V2 节点创建 **Leaf**（点击 Commit 节点上的 "Create Leaf" 或使用工具栏）
2. 选择 Leaf 类型为 **article**
3. 系统基于 Commit V2 的语义内容，自动生成策略研报
4. **添加 Constraints（约束条件）**：

   **Require（必须满足）**：
   - `require`: "SPY 20日收益率 < 0 且 VIX > 25 时降仓 30%"（市场状态过滤）
   - `require`: "组合目标波动率 12% 年化"（风控硬约束）
   - `require`: "单只股票最大权重 2%"（仓位约束）

   **Exclude（必须排除）**：
   - `exclude`: "无止损策略" — 理由：所有上线策略必须有风控机制
   - `exclude`: "无流动性过滤" — 理由：避免滑点失控

5. 确认生成 Leaf

### 你展示什么

- Leaf 节点出现在 Canvas 上，连接到 Commit V2
- **Article 内容**：自动生成的策略研报，结构化呈现策略要素
- **Constraints 列表**：require 和 exclude 约束清晰列出
- Leaf 是 Commit 的 **应用层输出**（Commit = 纯知识，Leaf = 约束 + 输出 + 验证）

### 你说什么

> "Leaf 是从版本化的知识中生成的应用层输出。这里我们生成了一份策略研报。"
> "关键是 Constraints — 这些是策略的硬约束条件。require 确保关键规则被遵守，exclude 确保危险模式被排除。"
> "这份研报不是凭空写的，它的每一条都追溯到 Commit V2 的语义条目。"

## Step 6：Pin 证据（来源可追溯）

### 6a. Pin Conversation（研究对话作为策略来源证据）

#### 你做什么

1. 在 Commit V2 或 Leaf 的侧边栏中找到 **Pin** 功能
2. 选择 **Pin Conversation** — 将研究对话 "Research: Momentum v1" Pin 为来源证据
3. 确认 Pin

#### 你展示什么

- Pin 关联显示在 Commit/Leaf 侧边栏中
- 点击 Pin 可以跳转到原始对话的具体位置
- 证据链：结论 → Pin → 原始对话

#### 你说什么

> "这个 Pin 把策略版本和原始研究对话关联起来了。复盘时，任何人都能一键定位到当时讨论的上下文。"

### 6b. Pin Leaf（策略研报作为输出证据）

#### 你做什么

1. 在 Commit V2 的侧边栏中，选择 **Pin Leaf** — 将刚才生成的策略研报 Pin 为输出证据
2. 确认 Pin

#### 你展示什么

- Commit V2 同时 Pin 了两种证据：
  - **Conversation Pin**：来源证据（策略从哪来）
  - **Leaf Pin**：输出证据（策略产出了什么）
- 完整证据链路：研究对话 → Commit V1 → 迭代对话 → Commit V2 → Leaf (研报)

#### 你说什么

> "现在这个策略版本有完整的证据链：从研究对话到语义版本，到最终研报。"
> "策略不是凭感觉，t3x 让每个版本都有证据链，复盘时一键定位。"

## 常见问题预备

**Q: 这跟写文档有什么区别？**
> 文档改了就没了。t3x 的每次修改都产生新的语义版本，V1/V2 永远可回溯，语义级别的 Diff 让你看到"到底改了什么"。

**Q: 能对接回测系统吗？**
> Leaf 的结构化输出（Sentences + Constraints）可以通过 API 导出，直接喂给回测引擎或执行 Agent。

**Q: 多人协作怎么处理？**
> 每个人在自己的 Branch 上迭代，用 Merge 功能合并，冲突会被语义级别检测到（比如"A 说做多、B 说做空"）。

**Q: 跟 Git 有什么关系？**
> t3x 是"语义层的 Git" — 用 SHA-256 哈希链保证不可篡改，用 DAG 支持分支和合并，但操作对象不是代码行，而是语义条目。
