# T3X 各页面功能缺失与新功能建议

> 基于 main 分支代码审查，按页面逐一分析当前已有功能、缺失功能、以及值得做的新功能。
> 重点关注：Share（分享）、Fork（复刻）、以及让 80% 的工具型用户用得爽的功能。

---

## 1. Chat 聊天页 (`/chat`, `/chat/[conversationId]`)

### 已有
- 流式对话、提取面板（YAML 预览）、可拖拽面板宽度

### 缺失
- **创建对话 API 未接通** — 代码里有 TODO: `"Wire to createConversation API in Task 7"`，目前只是跳转 URL
- 提取过程中没有进度指示（用户不知道 AI 正在提取）
- 提取结果没有 delta 持久化到 DB（下一步计划）
- 消息发送失败无错误提示

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分享对话快照 | 生成只读链接，查看者看到对话 + YAML 提取结果 | P1 |
| Fork 对话 | 在任意一轮消息处「从这里分叉」，保留上下文创建新对话 | P1 |
| 提取 Diff 高亮 | 每轮对话后，YAML 面板高亮变化："新增 3 条事实，修改 1 条" | P2 |
| 嵌入代码 | 生成 `<iframe>` 嵌入代码，开发者贴到文档/博客中展示提取结果 | P3 |

## 2. Project Canvas 画布页 (`/project/[projectId]`)

### 已有
- XYFlow 可视化图谱（对话/提交/叶子节点）、3 种视图（画布/时间线/列表）
- 右键菜单、视口持久化、30 秒自动刷新、新手引导清单

### 缺失
- **执行模式（Execution Mode）** — 明确标注 "Coming in v2.0"，当前是占位 UI
- 无撤销/重做
- 无画布内搜索
- 无批量操作（选中多个节点批量提交/删除）

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分享画布 | 公开只读画布链接，别人可以浏览你的知识图谱 | P1 |
| Fork 项目 | 一键复刻整个项目（所有提交、分支）— 开源增长的核心功能 | P0 |
| 画布标注 | 在节点上贴便签/评论，团队协作用 | P2 |
| 活动流 | 侧边栏显示最近变更："今天 3 个新提交"、"分支 X 已合并" | P2 |
| 从链接导入 | 粘贴分享链接或公开项目 URL，导入提交到画布作为新分支 | P2 |

## 3. Commit 提交详情页 (`/project/[projectId]/commit/[hash]`)

### 已有
- 多标签页（YAML / Graph / JSON / Relations）、Frame 索引、键盘导航
- 来源滑入查看器、操作侧边栏、溯源图

### 缺失
- Frame 图可视化交互有限（缩放/平移受限）
- Relations 标签页显示原始数据，无可视化
- 所有内容只读，无编辑能力
- 无法直接从提交详情跳到 Diff 比较

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分享提交卡片 | 漂亮的 OpenGraph 预览卡片，显示摘要、关键事实、置信度 | P1 |
| Fork 提交 | 把单个提交导入到自己的项目 | P1 |
| 导出格式 | 下载为 YAML / JSON / Markdown / CFPack | P1 |
| SDK 代码片段 | 页面上显示 `t3x.commits.get("sha256:...")` 代码，一键复制 | P2 |
| 与任意提交 Diff | "对比..." 下拉菜单，选择项目内任意提交进行 Diff | P2 |
| 嵌入组件 | `<script src="t3x.dev/embed/sha256:abc">` 渲染知识卡片 | P3 |

---

## 4. Leaf 叶子工作区 (`/project/[projectId]/leaf/[leafId]`)

### 已有
- 三区布局（来源/输出/检查器）、生成/展示两种模式
- 约束管理、断言验证、键盘导航、AI 建议约束
- 进度跟踪、模型选择、导出、从编辑中学习

### 缺失
- **语义验证未实现** — 代码明确标注 "Semantic validation is not yet supported"，当前仅支持精确匹配
- 无版本历史（看不到历次生成结果的对比）
- 无回滚到之前的输出版本
- 无法批量操作约束

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分享叶子输出 | 分享链接展示：输出文本 + 约束通过/失败状态 + 置信度。"这个 AI 输出是经过验证的" | P1 |
| Fork 叶子 | 克隆叶子的约束/配置到新叶子。"我要同样的质量规则但不同的内容" | P1 |
| 版本时间线 | 所有生成历史的可视化时间线，支持侧边栏对比 | P1 |
| 批量生成 | 同时用多个模型生成，网格对比结果，自动选断言通过率最高的 | P2 |
| Webhook 触发 | 生成成功后触发 webhook → Slack 通知 / 自动部署 | P3 |
| 一键存为模板 | 当前叶子配置直接保存为模板（不用去模板页） | P2 |

## 5. Diff 对比页 (`/project/[projectId]/diff`)

### 已有
- 双提交词级别 Diff、来源溯源、高亮显示

### 缺失
- 无 Diff 统计信息（新增多少句、修改多少、删除多少）
- 参数缺失时错误页没有引导用户如何创建 Diff
- 无法导出 Diff 结果
- 无法做三方对比（A vs B vs C）

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分享 Diff | "这是两个知识状态之间的变化"，可分享的 Diff 链接 | P1 |
| AI Diff 摘要 | 一键生成："新增 3 条用户偏好，解决 1 处矛盾，2 个置信度提升" | P1 |
| Diff 统计栏 | 顶部显示：+5 新增 / ~3 修改 / -2 删除 | P1 |
| Diff 批注 | 在具体变更处添加评论，类似 GitHub PR review | P2 |
| 导出 Diff 报告 | Markdown / PDF 格式，适合给团队/stakeholder 汇报 | P2 |

## 6. Merge 合并工作区 (`/project/[projectId]/merge/[mergeId]`)

### 已有
- 三方 Diff 可视化、冲突解决（选源/选目标/两者都要/自定义编辑）
- 来源溯源、合并成功庆祝动画

### 缺失
- 无法中途放弃合并
- 无冲突统计/复杂度指标（"本次合并有 5 个冲突，3 个简单 2 个复杂"）
- 无合并历史记录

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 合并请求（MR） | 正式的审批流程：创建 MR → 分配审阅者 → 审批后合并。知识的 "Pull Request" | P1 |
| 分享合并预览 | 合并前分享预览给团队审核 | P1 |
| AI 合并建议 | 每个冲突显示 AI 推荐及理由："建议保留 source，置信度更高（0.95 vs 0.72）" | P2 |
| 合并历史 | 查看历次合并及其解决方案，从过去决策中学习 | P2 |

## 7. Deploy 部署/评估页 (`/deploy`, `/deploy/eval/[runId]`, `/deploy/compare`)

### 已有
- Agent 注册管理、运行历史表、快速统计
- 评估报告（维度分数、轨迹时间线、断言审查）
- Pin + Re-tune 工作流、导出、分享、A/B 对比

### 缺失
- 无实时运行监控（只能看已完成的运行）
- 断言无筛选/排序
- 无 CSV/PDF 导出
- 无代理日志实时查看
- 无重复代理检测

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 实时运行监控 | 运行中显示实时进度、Token 消耗、当前步骤 | P1 |
| Fork 评估配置 | 克隆别人的评估设置（断言、约束、模型配置）到你的项目 | P1 |
| CI 集成 | GitHub Action：每次提交自动运行评估。README 徽章："T3X Knowledge Quality: 96%" | P2 |
| 成本追踪 | Token 使用趋势、每次生成成本、每个断言通过成本 | P2 |
| 排行榜 | 社区公开的模型性能排行榜 | P3 |

## 9. Share 分享页 (`/share/[token]`)

### 已有
- 只读查看器（叶子、运行、对比、提交）、无侧边栏的简洁布局

### 缺失
- 无过期时间显示（`expires_at` 字段存在但未使用）
- 无下载/导出按钮
- 无评论/批注能力
- 无链接创建时间显示

### 建议新功能
| 功能 | 说明 | 优先级 |
|------|------|--------|
| Fork 按钮 | **每个分享页都要有醒目的 "Fork 到我的 T3X" 按钮** — 开源知识增长的核心 | P0 |
| 过期控制 | 设置 TTL（1天/7天/30天/永久），前端显示倒计时 | P1 |
| 格式切换 | 查看者可切换 HTML（好看）/ YAML（原始数据）/ JSON（API 用） | P1 |
| 下载按钮 | 一键下载所选格式 | P1 |
| 权限级别 | 仅查看 / 可 Fork / 可评论 | P2 |
| 嵌入模式 | `?embed=true` 去掉外框，返回可嵌入的组件 | P2 |
| API 提示 | 分享页上显示 curl / SDK 代码片段 | P3 |

## 三大优先方向总结

### 1. Fork — 开源增长引擎（P0）

让知识像代码一样可以被 Fork。每个分享页都是一个增长入口。

```
分享页看到好东西 → 点 Fork → 导入到我的项目 → 在此基础上继续
```

- Fork 项目（完整复刻）
- Fork 提交（单个知识块）
- Fork 叶子（约束/配置复用）
- 分享页 Fork 按钮

### 2. Share 增强（P1）

当前 Share 只有基础的 token 链接 + 只读页面。需要：

- 过期控制（`expires_at` 已有字段，接通即可）
- 格式切换（HTML / YAML / JSON）
- 下载按钮
- 覆盖更多实体（对话、画布、Diff）

### 3. 每页体验打磨（P1-P2）

按用户旅程优先：

1. **Chat** — 提取进度指示、delta Diff 高亮
2. **Commit** — 导出格式、Diff 快捷入口
3. **Leaf** — 语义验证、版本历史
4. **Diff** — 统计栏、AI 摘要
5. **Deploy** — 实时监控、成本追踪
6. **Insights** — 增长图表、告警规则
